# 水族馆生物形象升级 & 动画特效增强 设计文档

**日期**: 2026-06-02
**状态**: 待实施

---

## 概述

对英语水族馆的 Canvas 2D 水族馆进行视觉和动画的全面升级。

### 两个核心目标

1. **生物形象升级**：从纯 Canvas 几何图形绘制 → 水彩手绘风格 PNG 精灵图
2. **动画特效增强**：6大全选（智能游动AI、粒子系统、动态光影、增强交互、环境细节、场景过渡）

---

## 一、生物精灵系统

### 1.1 素材规范

| 项目 | 规格 |
|------|------|
| 数量 | 15张，对应 `creatures.json` 中的15种生物 |
| 格式 | PNG（透明背景） |
| 推荐尺寸 | 200×200 ~ 400×400 px |
| 风格 | 统一水彩手绘风，梦幻海洋氛围 |
| 存放位置 | `static/sprites/` |
| 命名规则 | `<creature_id>.png`，如 `clownfish.png`、`dolphin.png` |

### 1.2 加载与渲染

- 新建 `static/js/sprites.js`，负责图片预加载和缓存
- 页面初始化时异步加载所有精灵
- 未加载完成时显示简化的几何图形占位（fallback）
- 加载完成后带淡入动画过渡
- 生物大小根据 `creatures.json` 中 `size` 字段缩放
- 稀有度特效：史诗/传说级生物绘制时叠加发光光晕

### 1.3 向后兼容

- `creatures.json` 数据结构不变，新增 `sprite_scale` 可选字段（默认1.0）
- 原本 `aquarium.js` 中的15个 `draw*` 方法（drawFish、drawDolphin 等）全部移除
- 新的绘制逻辑：`ctx.drawImage(spriteImage, x, y, w, h)` + 可选特效层

## 二、架构重构

当前 `aquarium.js`（~500行）拆分为 7 个独立模块：

```
static/js/
├── sprites.js        # 图片加载、缓存、绘制（~120行）
├── swim-ai.js        # 智能游动状态机（~200行）
├── particles.js      # 粒子特效管理器（~200行）
├── lighting.js       # 光影系统（~150行）
├── environment.js    # 海底环境装饰（~180行）
├── interaction.js    # 鼠标/触摸交互（~150行）
└── aquarium.js       # 主控制器（~150行，重构后）
```

`index.html` 中按依赖顺序引入。模块通过全局对象暴露接口（如 `Sprites`、`SwimAI`、`Particles`），由 `Aquarium` 主控制器统一调度。

## 三、智能游动 AI (swim-ai.js)

### 3.1 状态机

每个生物实例具备独立状态机：

```
自由游动 → 前往目标 → 短暂停留 → 转向 → 自由游动 ...
```

- **自由游动**：沿当前方向移动，速度在 min/max 范围内微调
- **前往目标**：在视野范围内随机选取目标点，easeInOut 加速/减速
- **短暂停留**：随机停留 1-4 秒，轻微摆动
- **转向**：触碰边界或到达目标时平滑转向（lerp 插值角度）

### 3.2 行为参数

不同生物类型有不同行为参数（在 `creatures.json` 或代码中配置）：

| 参数 | 说明 |
|------|------|
| `speed_range` | 游动速度范围 |
| `turn_rate` | 转向灵敏度 |
| `pause_chance` | 随机停留概率 |
| `schooling` | 是否参与集群行为 |

### 3.3 集群行为（鱼群）

- 同种生物 2-4 条可能形成小群体
- 群体内保持一定间距，朝同一大致方向移动
- 使用简化的 Boids 算法（分离 + 对齐 + 凝聚）

### 3.4 投喂反应

- 点击水面特定区域时，附近鱼群游向点击位置
- 持续 3-5 秒后恢复自由状态

## 四、粒子特效系统 (particles.js)

### 4.1 粒子类型

| 类型 | 描述 | 数量上限 |
|------|------|----------|
| 浮游光点 | 缓慢漂移的微小发光点，模拟微生物 | 30 |
| 气泡（升级） | 大小不一、摇晃上升、到达水面爆裂 | 25 |
| 游动尾迹 | 生物身后短暂残留的小气泡/光点 | 每生物 3-5 |
| 点击涟漪 | 点击水面产生扩散圆圈 | 不限（按需生成） |
| 盲盒烟花 | 抽到稀有/传说生物时的粒子爆发 | 不限（按需生成） |

### 4.2 粒子管理器

- 统一 `ParticleManager` 管理所有粒子生命周期
- 每帧更新位置、透明度、大小
- 生命周期结束自动回收
- 提供 `emit(type, x, y, options)` 统一接口

## 五、动态光影系统 (lighting.js)

### 5.1 水面焦散

- Canvas 顶部 30% 区域叠加焦散纹理
- 使用多个半透明椭圆/曲线模拟水面折射光斑
- 缓慢向下移动 + 左右摆动
- 夜晚时段焦散亮度降低

### 5.2 生物光晕

- 传说级生物：呼吸式光晕（半径正弦波 80%-120%）
- 史诗级生物：微弱恒定光晕
- 深海生物（发光水母）：自带荧光绿光晕

### 5.3 时间渐变

- 4个时间段切换从瞬间变为 60 秒平滑过渡
- 过渡期间所有颜色值（天空、水面、光线）线性插值
- 渐变在每一帧中逐步推进

### 5.4 深度雾

- 画面底部叠加淡蓝色雾层
- 增加水下纵深感和神秘感

## 六、环境装饰 (environment.js)

### 6.1 海草升级

- 当前：单段贝塞尔曲线摆动
- 升级：3-4 段骨骼式链条，每段独立摆动，更自然
- 增加海草颜色变化（深绿到浅绿）

### 6.2 新增装饰物

- **珊瑚礁**：随机 3-5 簇，不同颜色（粉、橙、紫），缓慢摇摆
- **海葵**：2-3 个，触手随水流晃动
- **小石头/贝壳**：散落在沙地上
- **沉船/宝箱**：1 个大型装饰物，增加探索感（可选）

### 6.3 鱼群装饰

- 远处偶尔出现一群快速游过的小鱼（非收集生物，纯装饰）

## 七、交互增强 (interaction.js)

### 7.1 点击生物

- 点击生物弹出精美信息卡（替代当前的简单 tooltip）
- 信息卡包含：生物图片、中英文名、稀有度标签、描述
- 弹出/关闭带缩放动画

### 7.2 鼠标涟漪

- 鼠标在水面区域（顶部 20%）移动时产生微弱波纹
- 点击任意位置产生扩散圆圈

### 7.3 拖拽生物

- 保持现有拖拽调整位置功能
- 增加拖拽时的半透明预览

## 八、场景过渡

- 时间切换从 `className` 瞬间切换改为 Canvas 颜色渐变
- `getTimeColors()` 返回值在帧间 lerp 插值
- 过渡时长：60 帧（约 1 秒 @60fps）
- 过渡期间生物活跃度也缓慢变化（夜晚游动更慢）

## 九、性能策略

### 9.1 目标

- 桌面端稳定 60fps
- 移动端稳定 30fps+

### 9.2 降级方案

- 移动端自动关闭焦散效果
- 粒子数量上限随设备像素比调整
- 使用 `requestAnimationFrame` 统一调度，避免重复绑定

### 9.3 调试

- 提供 `Aquarium.debug` 模式，显示 FPS 计数器
- 各特效可独立开关，便于性能定位
